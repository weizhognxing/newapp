# 生成列表接口文档

## 1. 基本信息

- 接口前缀：`/api/v1`
- 模块说明：当前“生成列表”对应后端 `divergence` 模块
- 认证方式：`Authorization: Bearer <token>`
- 后端实现：`backend/app/api/divergence.py`
- 前端调用：`frontend/src/api/divergence.js`

## 2. 接口清单

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取生成列表 | GET | `/divergence/list` | 不显式传租户路径时使用 |
| 按租户获取生成列表 | GET | `/divergence/tenant/{tenant_id}` | 前端当前优先使用 |
| 获取生成详情 | GET | `/divergence/{item_id}` | 查看单条生成记录详情 |
| 删除生成记录 | DELETE | `/divergence/{item_id}` | 删除单条记录及关联线程 |
| 保存生成结果 | POST | `/divergence/save` | 批量保存生成列表数据 |
| 查询生成进度 | POST | `/divergence/progress` | 按主题查询生成状态 |

## 3. 状态说明

`status` 字段含义如下：

| 值 | 含义 |
|----|------|
| 0 | 待生成 |
| 1 | 生成中 |
| 2 | 内容生成完成 |
| 3 | 生成内容失败 |
| 4 | 选题异常 |

说明：删除接口中，`status = 1` 的记录不允许删除。

## 4. 获取生成列表

### 4.1 通用列表接口

- 方法：`GET`
- 路径：`/api/v1/divergence/list`
- 说明：获取当前租户下的生成列表；如果请求参数中传入 `tenant_id`，优先使用该值，否则取当前登录用户的 `tenant_id`

请求参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| page_size | int | 否 | 20 | 每页数量 |
| tenant_id | int | 否 | 当前用户租户 | 租户 ID |
| user_id | int | 否 | - | 按创建人筛选 |

请求示例：

```http
GET /api/v1/divergence/list?page=1&page_size=20&tenant_id=1&user_id=12
Authorization: Bearer <token>
```

响应示例：

```json
{
  "success": true,
  "data": [
    {
      "id": 101,
      "tenant_id": 1,
      "user_id": 12,
      "status": 0,
      "note": "本次选题备注",
      "theme": "春季商超促销怎么做内容策划",
      "content": "围绕节日促销、爆品组合、门店活动展开",
      "keywords": "商超,促销,春季营销",
      "account_type_id": 2,
      "account_id": 18,
      "article_type": "1",
      "created_at": "2026-04-24T10:22:31"
    }
  ],
  "total": 56,
  "page": 1,
  "page_size": 20,
  "timestamp": "2026-04-24T10:25:12.123456"
}
```

响应字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| success | bool | 是否成功 |
| data | array | 列表数据 |
| total | int | 总条数 |
| page | int | 当前页码 |
| page_size | int | 每页数量 |
| timestamp | string | 响应时间 |

列表项字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 记录 ID |
| tenant_id | int | 租户 ID |
| user_id | int | 创建人 ID |
| status | int | 生成状态 |
| note | string | 备注 |
| theme | string | 主题/标题 |
| content | string | 生成内容 |
| keywords | string | 关键词 |
| account_type_id | int/null | 账号类型 ID |
| account_id | int/null | 账号 ID |
| article_type | string/null | 文章类型 |
| created_at | string | 创建时间，ISO 格式 |

状态码：

- `200`：成功
- `500`：查询失败

### 4.2 按租户获取生成列表

- 方法：`GET`
- 路径：`/api/v1/divergence/tenant/{tenant_id}`
- 说明：按路径中的租户 ID 查询生成列表；前端 `getDivergenceList` 在传入 `tenant_id` 时，当前实际优先调用该接口

路径参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | int | 是 | 租户 ID |

请求参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| page_size | int | 否 | 20 | 每页数量 |
| user_id | int | 否 | - | 按创建人筛选 |

请求示例：

```http
GET /api/v1/divergence/tenant/1?page=1&page_size=20&user_id=12
Authorization: Bearer <token>
```

响应示例与字段说明同 `4.1`。

状态码：

- `200`：成功
- `404`：租户或数据不存在时由权限/业务侧返回
- `500`：查询失败

## 6. 删除生成记录

- 方法：`DELETE`
- 路径：`/api/v1/divergence/{item_id}`
- 说明：删除单条生成记录，并同步删除 `threads` 表中关联的线程记录

路径参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| item_id | int | 是 | 生成记录 ID |

请求示例：

```http
DELETE /api/v1/divergence/101
Authorization: Bearer <token>
```

成功响应：

```json
{
  "success": true,
  "message": "删除成功",
  "deleted_threads": 3,
  "timestamp": "2026-04-24T10:35:00.000000"
}
```

失败响应示例：

```json
{
  "detail": "生成中的主题不允许删除"
}
```

状态码：

- `200`：成功
- `400`：记录处于生成中，不允许删除
- `404`：主题不存在
- `500`：删除失败

## 7. 保存生成结果

- 方法：`POST`
- 路径：`/api/v1/divergence/save`
- 说明：将 AI 发散结果或用户确认后的结果批量保存到生成列表

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tenant_id | int | 是 | 租户 ID |
| user_id | int | 是 | 用户 ID |
| items | array | 是 | 待保存主题列表 |
| note | string | 否 | 备注 |
| source_topics | string/null | 否 | 原始热点内容，JSON 字符串 |
| status | int | 否 | 生成状态，默认 `0` |
| account_type_id | int/null | 否 | 账号类型 ID |
| account_id | int/null | 否 | 账号 ID |
| article_type | string/null | 否 | 文章类型 |

`items` 元素结构：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| theme | string | 是 | 主题 |
| content | string | 是 | 详细内容 |
| keywords | string | 是 | 关键词 |

请求示例：

```json
{
  "tenant_id": 1,
  "user_id": 12,
  "status": 0,
  "note": "本次测试保存",
  "account_type_id": 2,
  "account_id": 18,
  "article_type": "1",
  "source_topics": "[{\"title\":\"热点标题A\",\"summary\":\"热点摘要A\",\"url\":\"https://example.com/a\",\"source\":\"微博\",\"created_at\":\"2026-04-24 09:00:00\"}]",
  "items": [
    {
      "theme": "春季商超促销怎么做内容策划",
      "content": "围绕节日促销、爆品组合、门店活动展开",
      "keywords": "商超,促销,春季营销"
    }
  ]
}
```

成功响应：

```json
{
  "success": true,
  "message": "成功保存 1 条发散主题",
  "count": 1,
  "status": 0,
  "timestamp": "2026-04-24T10:40:00.000000"
}
```

状态码：

- `200`：成功
- `500`：保存失败

## 8. 查询生成进度

- 方法：`POST`
- 路径：`/api/v1/divergence/progress`
- 说明：根据主题列表查询当前生成状态汇总

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| themes | array[string] | 是 | 主题列表 |

请求示例：

```json
{
  "themes": [
    "春季商超促销怎么做内容策划",
    "门店活动如何结合短视频推广"
  ]
}
```

响应示例：

```json
{
  "success": true,
  "data": [
    {
      "theme": "春季商超促销怎么做内容策划",
      "status": 2,
      "content": "正文内容...",
      "keywords": "商超,促销"
    }
  ],
  "summary": {
    "total": 1,
    "pending": 0,
    "generating": 0,
    "completed": 1,
    "failed": 0,
    "topic_exception": 0
  },
  "timestamp": "2026-04-24T10:45:00.000000"
}
```

汇总字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| total | int | 总数 |
| pending | int | 待生成数量 |
| generating | int | 生成中数量 |
| completed | int | 已完成数量 |
| failed | int | 失败数量 |
| topic_exception | int | 选题异常数量 |

状态码：

- `200`：成功
- `500`：查询失败

## 9. 前端实际调用说明

前端文件：`frontend/src/api/divergence.js`

- `getDivergenceList(params)`
  - 当 `params.tenant_id` 为有效数字时，请求 `GET /divergence/tenant/{tenant_id}`
  - 否则请求 `GET /divergence/list`
- `getDivergenceDetail(itemId)` 对应 `GET /divergence/{item_id}`
- `deleteDivergence(itemId)` 对应 `DELETE /divergence/{item_id}`
- `saveDivergence(params)` 对应 `POST /divergence/save`
- `getProgress(themes)` 对应 `POST /divergence/progress`

## 10. 联调注意事项

- 列表接口当前仅支持 `page`、`page_size`、`tenant_id`、`user_id` 参数，不支持按标题、状态、时间范围筛选
- 列表接口返回字段为 `theme`，详情接口返回字段为 `title`，前端展示时需要做字段统一
- 删除接口会同步删除 `threads` 表内 `theme_id` 关联数据
- 详情接口会校验租户访问权限，跨租户访问会被拦截
- `source_topics` 在保存时为 JSON 字符串；详情返回时会被解析为数组结构
