# 发布列表与历史选题接口文档

## 1. 基本信息

- 接口前缀：`/api/v1`
- 认证方式：`Authorization: Bearer <token>`
- 发布列表后端实现：`backend/app/api/threads.py`
- 发布列表前端调用：`frontend/src/api/threads.js`
- 历史选题后端实现：`backend/app/api/historical_planning.py`
- 历史选题前端调用：`frontend/src/api/historicalPlanning.js`

## 2. 接口清单

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取发布列表 | GET | `/threads/publish-records/list` | 查询当前租户下的发布记录列表 |
| 获取历史选题批次列表 | GET | `/historical-planning/sessions` | 分页查询历史选题批次 |
| 获取历史选题批次详情 | GET | `/historical-planning/session/{planning_session_id}` | 查询某个历史选题批次下的明细结果 |

## 3. 发布状态说明

发布列表中的 `status` 字段含义如下：

| 值 | 含义 |
|----|------|
| 0 | 待发布/处理中 |
| 1 | 发布成功 |
| 2 | 发布失败 |
| 3 | 改写失败 |
| 4 | 改写中 |

`publish_state` 查询参数含义如下：

| 值 | 含义 |
|----|------|
| all | 全部 |
| publishing | 发布中，实际筛选 `status IN (0, 4)` |
| completed | 已完成，实际筛选 `status = 1` |
| failed | 已失败，实际筛选 `status IN (2, 3)` |

## 4. 获取发布列表

- 方法：`GET`
- 路径：`/api/v1/threads/publish-records/list`
- 说明：按租户分页查询发布记录，返回发布平台、文章标题、账号、发布时间、失败原因等信息

请求参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| tenant_id | int | 是 | - | 租户 ID，用于权限校验 |
| skip | int | 否 | 0 | 分页偏移量，从 0 开始 |
| limit | int | 否 | 20 | 每页数量，最大 100 |
| publish_state | string | 否 | all | 发布状态筛选，支持 `all/publishing/completed/failed` |

请求示例：

```http
GET /api/v1/threads/publish-records/list?tenant_id=1&skip=0&limit=20&publish_state=all
Authorization: Bearer <token>
```

列表项字段：

- `200`：成功
- `400`：`publish_state` 不支持
- `401`：未登录或登录态失效
- `403`：无权访问该租户数据
- `500`：查询失败

## 5. 获取历史选题批次列表

- 方法：`GET`
- 路径：`/api/v1/historical-planning/sessions`
- 说明：分页查询历史选题批次，用于“历史选题”列表页展示

请求参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码，从 1 开始 |
| page_size | int | 否 | 20 | 每页数量 |
| tenant_id | int | 否 | 当前登录用户 tenant_id | 租户 ID；管理员可显式指定 |
| user_id | int | 否 | - | 按创建用户筛选 |

请求示例：

```http
GET /api/v1/historical-planning/sessions?page=1&page_size=20&tenant_id=1&user_id=12
Authorization: Bearer <token>
```

## 7. 补充说明

- 发布列表接口的租户权限通过 `resolve_request_tenant_id` 校验，非管理员通常只能访问自身租户数据。
- 历史选题接口在未传 `tenant_id` 时，会默认使用当前登录用户的 `tenant_id`。
- 历史选题列表中的 `source_topics_preview` 只返回前 2 条来源数据；完整来源数据需要查看详情接口中的 `session.source_topics`。
- 历史选题详情结果按 `result_order ASC, id ASC` 排序返回。
